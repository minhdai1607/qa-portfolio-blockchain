# Test Plan – Web Application (Portfolio)
Note:
This test plan is a sample document created for portfolio demonstration.
It does not represent any real company project.

## 1. Introduction
This test plan describes the testing approach for a web application.
This document is created for portfolio purposes only.

## 2. Scope
### In Scope
- User authentication (Google Sign-in)
- Core user flows
- Error handling

### Out of Scope
- Third-party system testing
- Production data validation

## 3. Test Approach
- Manual testing
- Automation testing (Playwright – concept)
- Regression testing before release

## 4. Test Types
- Functional
- Integration
- Security
- UI/UX
- Regression

## 5. Test Environment
- Web browsers: Chrome, Firefox
- OS: Windows, macOS

## 6. Entry & Exit Criteria
### Entry Criteria
- Requirements approved
- Test environment ready

### Exit Criteria
- All critical test cases passed
- No open blocker defects

## 7. Risks & Mitigation
| Risk | Mitigation |
|------|------------|
| Unstable third-party login | Retry mechanism & monitoring |

## 8. Deliverables
- Test cases
- Bug reports
- Test summary report
